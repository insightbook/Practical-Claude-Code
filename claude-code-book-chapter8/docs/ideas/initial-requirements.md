# 태스크 관리 CLI 도구 아이디어

## 배경 및 문제

개발자가 터미널에서 작업하는 도중 태스크 관리를 위해 브라우저나 GUI 애플리케이션으로 전환하는 것은 비효율적이다. 특히 프리랜서나 여러 프로젝트를 병행하는 엔지니어에게 컨텍스트 전환은 생산성 저하로 직결된다.

## 컨셉

**Devtask**: 개발자를 위한 CLI 기반 태스크 관리 도구

- 터미널을 벗어나지 않고 태스크를 일괄 관리
- 간결하고 빠른 조작
- Git 연동을 통한 개발 흐름 통합
- AI 기반 우선순위 자동 추정으로 의사결정 지원

## 주요 기능

### 태스크 관리 기본 기능

- 태스크 생성, 편집, 삭제 (CRUD)
- 상태 관리 (미착수 / 진행 중 / 완료 / 보관)
- 마감일 설정 및 표시
- 태스크 상세 조회

### CLI 조작

```bash
# 기본 사용법
devtask init myproject      # 프로젝트 초기화
devtask add "기능 개발"      # 태스크 추가
devtask list                # 목록 표시
devtask show <id>           # 상세 보기
devtask update <id> --status done  # 상태 업데이트
devtask delete <id>         # 삭제
```
### 우선순위 자동 추정

다음 요소를 종합하여 우선순위를 산출한다.
- 마감까지 남은 시간 (짧을수록 높은 우선순위)
- 태스크 생성 후 경과 시간 (길수록 높은 우선순위)
- 현재 상태 (진행 중이면 높은 우선순위)

가중치를 조정할 수 있어 팀 또는 프로젝트 특성에 맞게 설정이 가능하다.

### Git 연동

- 현재 브랜치에 연결된 태스크 관리
- 태스크 시작 시 브랜치 자동 생성
- 브랜치명 자동 생성 및 정규화

### 데이터 저장

- JSON 파일로 로컬 저장
- 프로젝트 디렉터리 아래의 `.devtask/`에 저장
- 자동 백업 기능

## 비기능 요건

- 명령어 실행 시간 < 100ms
- JSON 파일 크기 < 1MB (태스크 1,000개 기준)
- Node.js 18 이상 동작
- 크로스 플랫폼 지원 (macOS, Linux, Windows)
